# 金融建账业务规则

## 规则1：账户创建规则

### 触发条件
当用户提交建账申请时，系统需要自动创建基础账户结构。

### 执行动作
1. 创建主账户（Main Account）
2. 创建子账户（Sub Account）
3. 初始化账户余额为0
4. 设置账户状态为"激活"

### 配置参数
- 账户类型：支持对公账户、对私账户
- 货币单位：CNY、USD、EUR
- 账户编码规则：6位数字编码

### 业务场景
适用于新客户开户、存量客户新增业务线等场景。

### 注意事项
- 同一客户只能创建一个对公账户
- 对私账户数量不超过5个
- 账户编码需唯一

---

## 规则2：转账流程规则

### 触发条件
当用户发起转账请求时，系统需要执行转账逻辑。

### 执行动作
1. 验证发起方账户余额是否充足
2. 检查接收方账户是否存在且状态正常
3. 执行账户余额扣减和增加
4. 生成转账流水记录
5. 发送转账成功通知

### 配置参数
- 单笔转账限额：10,000 CNY
- 每日转账限额：50,000 CNY
- 转账手续费率：0.1%

### 业务场景
适用于账户间资金划转、费用支付、收入归集等场景。

### 注意事项
- 转账金额必须大于0
- 转账双方账户不能相同
- 转账失败需自动回滚

---

## 规则3：账户冻结规则

### 触发条件
当账户出现异常情况时，系统需要冻结账户。

### 执行动作
1. 设置账户状态为"冻结"
2. 暂停账户所有交易功能
3. 发送冻结通知给账户持有人
4. 记录冻结原因和时间

### 配置参数
- 冻结原因类型：异常交易、风险控制、合规要求
- 冻结时长：临时冻结（24小时）、永久冻结（需人工审核解冻）

### 业务场景
适用于账户风险控制、异常交易处理、合规审查等场景。

### 注意事项
- 冻结前需发送预警通知
- 临时冻结到期后自动解冻
- 永久冻结需人工审核后解冻
